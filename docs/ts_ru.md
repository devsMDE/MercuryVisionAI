# MercuryVision Studio 2.1 – Техническое задание (ТЗ)

## 1. Введение
### 1.1 Назначение
Настоящий документ является Техническим заданием (ТЗ) платформы MercuryVision Studio — премиальной веб-системы на базе ИИ, предназначенной для геопространственной разведки и выявления изменений (Change Detection).

### 1.2 Область применения
Описывается архитектура ПО, премиальные стандарты дизайна (уровень «Apple»), внедрение высшего уровня безопасности и методы серверной интеграции с использованием PHP, Firebase и моделей ChangeFormer.

## 2. Архитектура
### 2.1 Фронтенд (Клиентская часть)
- **Технологии:** HTML5, Vanilla JavaScript (ES6+), CSS3.
- **Парадигма UI:** Глассморфизм (Glassmorphism), динамически адаптивная сетка, плавные микроинтеракции.
- **Иконки:** Отрисовка через систему Lucide.

### 2.2 Бэкенд (Серверная часть)
- **Среда выполнения:** PHP 8+ для маршрутизации и валидации API.
- **База данных:** Легкое локальное хранилище данных на основе JSON (`api/json_db.php`), имитирующее PDO для максимальной портативности, совместно с Firebase Realtime Database.
- **Аутентификация:** Google Firebase Auth (JWT токены на клиенте и сессии на сервере).

## 3. Основной функционал
- **AI Compare:** Общий поиск визуальных отличий на снимках.
- **Специализированные режимы:** Водная динамика, Лесная аналитика, Индекс NDVI / Сельское хозяйство, Городская аналитика.
- **Стратегический ИИ-Аналитик (Чат):** Контекстно-зависимая боковая панель с историей, генерирующая текстовые инсайты на основе метрик анализа.

## 4. Требования к UI/UX дизайну
- **Плавность интерфейса:** Каждый кликабельный или наводимый элемент должен использовать кривые Безье (`cubic-bezier(0.16, 1, 0.3, 1)`) для анимации.
- **Премиальная эстетика:** Исключение плоских однотонных цветов. Многоуровневые градиенты, глубокие мягкие тени (`box-shadow`), отсутствие резких углов.

## 5. Защита и Безопасность
- **Хранение секретов:** Все кастомные API ключи пользователей (OpenAI/Gemini) хранятся исключительно в `localStorage` и не передаются в открытом виде.
- **CORS политика:** Ограничение разрешенных доменов в `config.php`.
- **Защита от XSS и инъекций:** Полное экранирование данных перед выводом на фронтенде и строгая маршрутизация запросов.

## 6. Целевая аудитория
Аналитики, исследователи, руководители уровня Enterprise, которым требуется надежный инструмент для сравнения топографических или спутниковых снимков в форматах «до» и «после».
